## 2. 기능 구현 방식 관련

### 2.1 인증 토큰 관리

**질문**: Access Token과 Refresh Token을 어떻게 저장하고 관리할까요?

**옵션**:

#### Access Token 저장 위치
- [ ] **localStorage**: 브라우저 종료 후에도 유지, XSS 공격에 취약
- [ ] **sessionStorage**: 브라우저 탭 종료 시 삭제, XSS 공격에 취약
- [ ] **메모리 (state)**: 페이지 새로고침 시 삭제, 가장 안전
- [ ] **httpOnly Cookie**: XSS 공격에 안전, CSRF 공격에 취약

#### Refresh Token 저장 위치
- [ ] **httpOnly Cookie**: 가장 안전, CSRF 방지 필요
- [ ] **localStorage**: 구현 간단, XSS 공격에 취약

**권장 사항**: 
- Access Token: 메모리 또는 localStorage
- Refresh Token: httpOnly Cookie (CSRF 토큰과 함께 사용)

**확인 필요**: 
- [ ] 토큰 갱신 전략? (자동 갱신 vs 수동 갱신)
- [ ] 토큰 만료 시 자동 로그아웃할지?
